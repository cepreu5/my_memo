# План: Интернационализация на my_memo — ARB + Автоматичен превод + Редактор

## Концепция

1. **ARB файлове** — стандартен Flutter i18n подход
2. **Настройки → Превод на интерфейса** — машинен превод на всички низове, генерира ARB файл
3. **Настройки → Редакция на интерфейса** — ръчен редактор показва всеки низ (базов + превод), позволява ръчна корекция
4. **Базов език**: текущият език на интерфейса (първоначално български). Потребителят може да избере друг базов език, от който да се превежда

## Фаза 4: Сервиз за машинен превод

### `lib/translation_service.dart`

```dart
import 'dart:convert';
import 'package:http/http.dart' as http;

class TranslationService {
  // Google Translate API (без ключ — безплатен tier)

  static Future<Map<String, String>> translateAll({
    required Map<String, String> sourceStrings,
    required String targetLanguage,   // Код на целевия език (напр. "en", "de")
    required String sourceLanguage,   // Код на базовия език (напр. "bg")
  }) async {
    final translations = <String, String>{};

    for (final entry in sourceStrings.entries) {
      final translated = await _translateText(
        entry.value,
        sourceLanguage,
        targetLanguage,
      );
      translations[entry.key] = translated;
    }

    return translations;
  }

  static Future<String> _translateText(
    String text,
    String sourceLang,
    String targetLang,
  ) async {
    // Google Translate free API
    final url = Uri.parse(
      'https://translate.googleapis.com/translate_a/single'
      '?client=gtx&sl=$sourceLang&tl=$targetLang&dt=t&q=${Uri.encodeComponent(text)}',
    );

    final response = await http.get(url);
    if (response.statusCode == 200) {
      final data = json.decode(response.body);
      return data[0][0][0] as String;
    }
    return text; // Fallback на оригинала
  }
}
```

## Фаза 5: Екран "Превод на интерфейса"

### `lib/translation_screen.dart`

**UI**:
```
┌─────────────────────────────┐
│ Превод на интерфейса        │
├─────────────────────────────┤
│ Базов език: [Български ▼]   │  ← текущият език по подразбиране
│ Целеви език: [Английски ▼]  │
│                             │
│ [Преведи от БГ към EN]     │
│                             │
│ Прогрес: 45/200 низа        │
│ ████████░░░░░░░░ 22%        │
│                             │
│ [Запази ARB файл]           │
└─────────────────────────────┘
```

**Логика**:
1. Открива кой е текущият език (по подразбиране български)
2. Потребителят може да промени базовия език (напр. ако вече има EN превод, може да превежда от EN към DE)
3. Потребителят избира целеви език
4. Прави batch превод на всички низове от базовия към целевия
5. Показва прогрес
6. Записва новия ARB файл

## Верификация

1. `flutter pub get`
2. `flutter gen-l10n` — генериране на локализации
3. `flutter analyze` — без грешки
4. `flutter build apk` — успешен build
5. Тестове:
   - Смяна на език в настройките
   - Всички низове се превеждат коректно
   - Автоматичният превод работи (нужен интернет)
   - Ръчният редактор записва промените
   - Фолбек към базов език при липсващ превод

---

## Рискове

- Google Translate free API може да има rate limits
- Качеството на машинния превод варира
- Нужен е интернет при автоматичен превод
- ~200+ низа за превод — ръчната редакция отнема време
