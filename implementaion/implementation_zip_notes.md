# Локален Backup/Restore на бележки чрез ZIP архив

Потребителят иска възможност да запазва и възстановява бележките си от ZIP архив — подобно на вече съществуващия бекъп на снимки, но работещ **локално** (без нужда от Google Drive).

## Предложени промени

### Как ще работи

**Експорт (Backup):**
1. Чете всички бележки от SQLite базата
2. Сериализира ги в JSON формат
3. Включва всички асоциирани снимки (от `imagePath`) в архива
4. Създава ZIP файл `MyMemo_Notes_Backup.zip` в Downloads папката
5. Показва съобщение с броя на експортираните бележки

**Импорт (Restore):**
1. Потребителят избира ZIP файл (file picker)
2. Парсва JSON файла с бележките от архива
3. Извлича снимките в app documents директорията
4. За всяка бележка проверява UUID — ако вече съществува, обновява я; ако не — вмъква нова
5. Показва съобщение с резултата

> [!IMPORTANT]
> **File Picker**: В момента няма `file_picker` dependency в проекта. Ще трябва да добавим `file_picker` пакета за да може потребителят да избере ZIP файл за импорт. Алтернатива — да ползваме фиксиран път (Downloads), но file picker е по-удобен.
